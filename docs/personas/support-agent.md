# Persona: Customer Support Agent

## Profile
- **Name:** Priya Patel
- **Role:** Senior Support Agent
- **Age:** 27
- **Tech comfort:** High — fast with keyboard shortcuts, multi-tabs, CRM tools
- **Tickets/day:** 40–60
- **Time in Maropost:** 6+ hours/day (primary work tool)

## Daily Tasks in Maropost (by frequency)

1. **Triage tickets** — scan new tickets, assign priority, categorize (continuous)
2. **Look up contacts** — find customer by email, check order history, past interactions (every ticket)
3. **Check order status** — verify fulfillment and payment status for customer inquiries (every ticket)
4. **Respond to tickets** — use reply templates, add internal notes (continuous)
5. **Escalate issues** — flag orders with problems, reassign tickets to specialists (as needed)
6. **Update contact info** — fix email addresses, add tags for follow-up (as needed)
7. **Review ticket metrics** — response time, resolution rate, CSAT (end of day)

## Key Pages Used

| Page | Frequency | What she does |
|------|-----------|--------------|
| Tickets | Continuous | Main workspace — view, respond, resolve |
| All Contacts | Every ticket | Look up customer details, order history |
| Sales Orders | Frequent | Check order fulfillment/payment status |
| Contact Tags | Weekly | Tag customers for follow-up campaigns |
| Settings > Service | Occasional | Manage reply templates, ticket types |

## Pain Points

1. **Context switching is slow** — has to open Tickets, then navigate to Contacts, then to Orders to understand one customer issue. Wants a unified customer view
2. **No ticket-to-order linking** — can't see which order a ticket is about without manually searching
3. **Reply templates are basic** — wants variables (customer name, order number) auto-filled
4. **No keyboard shortcuts** — wants Cmd+K command palette for fast navigation
5. **Ticket list doesn't show customer info** — just shows subject line, no customer name or order context in the list view
6. **Can't bulk-close resolved tickets** — has to close one by one

## Success Metrics
- First response time < 2 hours
- Resolution rate > 85%
- CSAT score > 4.5/5
- Zero tickets falling through cracks (no "forgotten" open tickets)

## Interface Expectations
- Split-pane layout: ticket list on left, detail on right (like email clients)
- Customer context panel showing order history, past tickets, contact details
- Quick-reply with template variables
- Keyboard-navigable (Tab through tickets, Enter to open, Esc to close)
- Status color coding: red = urgent, orange = waiting, green = resolved
- Notification badge for new tickets and replies

## Quote
> "Every extra click costs me time, and time means a customer waiting longer. I need to see the customer's full picture without leaving my ticket queue."
